Atlas V2

New Features


  • Real Google Gemini AI Integration: Replaces simulated responses with genuine Google Gemini 2.5 models (Flash, Pro, Flash-Lite), enabling real AI-generated content with advanced reasoning.
  • Advanced Multi-Language Text-to-Speech: Detects language using Unicode range scanning (Bengali, Hindi, English), selects appropriate neural voices when available, splits text into sentences for natural pacing, and cancels previous speech before starting new output.
  • Modern Modal Settings Panel: Replaces the basic dropdown with a full-screen modal featuring blur backdrop, smooth animations, keyboard navigation (Escape to close), and click-outside dismissal.
  • Animated Theme Slider with Circular Reveal: Introduces a sliding toggle with animated sun/moon icons and a circular ripple effect triggered from the click point during theme transitions.
  • Functional Model Selector: Model selection now directly determines the Gemini endpoint used for API requests, enabling real control over speed and capability.
  • Permanent Assistant Labeling: Replaces dynamic timestamps with a static “Atlas” label to clearly identify assistant messages and strengthen branding.

Reported Problems (Bugs for V3 Consideration)


  • No Streaming Responses: Uses non-streaming generateContent endpoint, requiring users to wait for full responses instead of real-time output.
  • No Error Recovery for API Failures: Displays a generic error message without retry options or fallback models.
  • Image Handling Lacks Proper MIME Type Detection: Hardcodes mime_type as "image/jpeg", which may fail for PNG or WebP uploads.
  • Conversation History Truncation Risk: Sends full message history each request without token management, risking context overflow.
  • Regenerate Function Creates Duplicate History: Message slicing may cause UI artifacts or duplicate typing indicators during regeneration.
  • No Loading State for Image Attachments: FileReader processing has no progress indicator, causing perceived UI freeze.
  • Theme Animation Performance Issue: Circular reveal animation may stutter on lower-end devices due to heavy rendering.
Atlas V1 [RELEASED ON 15TH MARCH, 2026]

Features


  • Puter.js Authentication System: Login screen with loading animation using Puter authentication before accessing chat.
  • Light/Dark Mode Toggle: Theme switcher (lightbulb icon) with locally saved user preference.
  • Multi-Modal Input Options: "+" menu supporting image upload from gallery and direct camera capture.
  • Simulated AI Chat Interface: User/assistant bubbles, typing indicators, timestamps, and action buttons (copy, text-to-speech, regenerate).
  • Rich Content Rendering: Markdown, LaTeX (MathJax), tables, and image previews inside chat.
  • Responsive Mobile Design: Touch-optimized layout, larger tap targets, full-screen mobile support, reduced tap highlights.

Reported Problems


  • Authentication Fails Silently: Session validation fails without error handling or fallback, causing login hang issues.
  • Inconsistent Session Handling: Expired or failed sessions do not properly update UI state, leading to silent API failures.
  • AbortController Mismanagement: Stopped streams do not update message state correctly, causing incomplete message handling and ordering issues.
  • Image Attachment State Bug: Image variable does not always reset, sometimes sending previous images unintentionally.
  • Regenerate Function Breaks with Images: Regeneration loses image context and only resends text.
  • Message Order Corruption During Streaming: Race condition allows messages to appear in incorrect order.
  • Text-to-Speech Overlap: Multiple speech instances run simultaneously without canceling previous audio.
  • Model Selector Has No Effect: Model dropdown does not affect actual API calls, making it non-functional.
